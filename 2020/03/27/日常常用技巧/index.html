ubuntu配置v2ray、Qv2ray客户端

  1. 下载安装Qv2ray
  2. 下载安装v2ray-core

注意问题：

  • Extract the v2ray core files into a fixed path. It is suggested to extract the files into $QV2RAY_CONFIG_PATH/vcore, where $QV2RAY_CONFIG_PATH is the directory where Qv2ray store it’s data.
  • 访问 http://raw.githubusercontent.com/ 时，提示无法访问
    • 编辑 etc/hosts 文件，新增下列内容或者设置https_proxy

  1. 运行 service v2ray start 来启动 V2Ray 进程；
    之后可以使用 service v2ray start|stop|status|reload|restart|force-reload 控制 V2Ray 的运行。
    或者使用

  2. 设置系统代理

  • 全局代理：在.bashrc添加

  • 浏览器局部代理，直接在浏览器network proxy手动添加代理

注：（第一次用以下方法成功了，以后就搞不成了，很奇怪）

  1. v2ray官网手册 上提供了多种下载方法的说明。推荐直接在github上的 v2ray-core项目 中，按照自己的系统，下载最新的 release 版本压缩包。解压。
  2. 在本地配置v2ray客户端前，请先确保已经成功在墙外服务器配置好v2ray的服务器端,按照墙外服务器的相关配置，参考 v2ray官网手册 修改解压缩目录下的config.json文件。推荐如果正在使用win系统版的v2rayN软件，可以直接从中导出配置文件，替换掉解压缩出的config.json，不用修改即可直接使用。
  3. /usr/local目录下，然后进入该目录，创建服务端配置文件目录 将客户端配置文件/v2ray/config.json复制到刚才创建的目录
  4. 启动v2ray服务
  5. 设置开机自启
  6. 检查v2ray启动状态，检查端口是否被占用
  7. 开启代理
    配置完v2ray后，还需要让linux的各种下载服务使用代理，才可以实现翻墙。设置代理有配置环境变量和配置proxychains两种方案，推荐使用proxychains方案。首先安装proxychains
  8. v2ray默认PAC模式，google使用switchyOmega插件设置
  9. 如果是只代理一个程序的话可以采用proxychains进行代理，在终端运行程序前加上proxychains 比如proxychains google-chrome ,那么就可以仅代理这个程序的网络，默认配置文件在/etc/proxychains.conf ，添加来自v2ray的http或者socks即可。在debian的官方源里有，ubuntu应该也有，详细可以看官方文档。
